# Echo-Quill Alchemist · 回声炼金术士

把一本小说"提炼"成系统的世界观与文风，之后让 Claude 在那个世界里替你续写——记得人物、地点、最近发生的事。全程跟 Claude 说话，不需要自己跑命令。

---

## 用起来是什么样

你只跟 Claude 对话，下面这些话它都听得懂：

### 训练 · 把一本小说"喂"给系统

```
你 ▸  用 echo-quill 训练 D:/novels/三体.txt
```

Claude 会自检环境、提示一下大致 token 成本、打开仪表盘，然后开始一片一片地训练。仪表盘里可以实时看进度。

### 续写 · 带记忆地写下去

```
你 ▸  续写一段：他推开门，屋里很暗
```

系统会调出训练过的角色卡、相关地点、最近发生过的事件，按学到的风格写一段（300~800 字）。每次生成都会自动归档到 `core/data/drafts/`，方便你回头找。

不满意？直接告诉 Claude 怎么改：

```
你 ▸  把李四改成女性，重写一遍
```

### 续写完想留下，把这一篇也"喂"给系统

```
你 ▸  把刚才那篇训练进去
```

那篇续写会作为 **user-layer** 单独存一份。之后再续写时，系统就知道这一段"已经发生过了"，不会跟它矛盾。

### 想回到只剩原著的状态

```
你 ▸  回滚，只保留原著
```

你训练的所有续写会被移到 `archive/` —— 不是真删，万一后悔了还能恢复回来。

### 也可以让它"忘掉人物只学风格"

如果你不想要记忆、只想纯模仿语感：

```
你 ▸  用纯风格模式续写：夜半，他推开窗
```

### 其他能听懂的话

- "show me the dashboard / 打开仪表盘"
- "how's training going / 现在进度怎么样"
- "停一下服务 / shut it down"
- "列出训练过哪些 / show corpora"
- "把刚才回滚的那篇恢复回来"
- "解释下这玩意儿怎么工作的"
- "出错了 / something broke"

## 安装

```bash
npx skills add hoshyo/echo-quill-alchemist
```

之后跟 Claude 说话就行。第一次训练前 Claude 会一次性问你是否要装依赖（pip 包 ~2 GB）以及拉两个本地模型（MiniLM + bge-m3，加起来 ~2.4 GB），都点头后才开始训练 —— 不会训练到一半再卡住下载。后续训练 / 续写直接复用本地缓存。

需要一个能调通 Anthropic Messages API 或 OpenAI Chat Completions 的 endpoint。如果你在用 [CC Switch](https://github.com/farion1231/cc-switch) 管理 Claude Code 的 endpoint，Echo-Quill 会自动继承，**不用额外配 `.env`**。

## 系统是怎么工作的（一句话版）

一本小说被切成 `(上文, 真实续写)` 的窗口，每个窗口同时做四件事：

1. **学风格** —— LLM 生成 N 个候选 + 1 个故意写烂的"反例"，用本地 MiniLM + ROUGE-L 给候选打分，强制产出风格偏好对（DPO 数据）
2. **学世界观** —— LLM 抽取出现的人物、地点、物件，按别名合并去重，存成结构化卡片
3. **学情节** —— LLM 抽取关键事件，按时间堆成事件链
4. **熬"风格规则"** —— 在线提炼语感规则，规则有寿命，跟得上原文的活下来，跟不上的被裁

当你之后让 Claude 续写时，系统会先按你的上文检索相关的 **风格规则 + 角色卡 + 场景物件 + 最近事件 + 高分示例**，拼成 5 块提示词喂给 LLM 再让它写。

### 训练单元 = "bundle"，所以回滚很干净

每次训练都产出一个独立目录：

```
core/data/corpora/
  original/<sha[:12]>/   ← 原著训练的产物
  user/<时间戳_随机>/    ← 你训练进去的某篇续写
  ...
```

每个目录包含那次训练学到的**全部**东西（规则 / 人物卡 / 事件 / DPO 偏好对）。

- "回滚到只剩原著" = 把所有 `user/*` 目录移到 `archive/`
- "撤销某次 user 训练" = 把那一个目录移到 `archive/`
- "恢复刚才回滚的那篇" = 从 `archive/` 移回来

**永远是移动，不是删除**。你可以反复回滚 / 恢复，不会丢数据。除非你显式 `--wipe-all`，那才是真删。

## 三条不可妥协的设计原则

- **大模型不当裁判**：所有质量分都是 MiniLM 余弦 + ROUGE-L F1 的加权和；LLM 只负责"生成"和"按 JSON schema 抽取结构化信息"，永远不让它打 0-10 分
- **每个窗口必产偏好数据**：永远生成一个困难负样本兜底，差距够大时再产第二对，让"风格 DPO 数据集"永远在增长
- **状态都有寿命 / 出处 / 可回滚**：风格规则有 lifespan + hit_count，每个人物/事件都打 `source` 与 `corpus_id` 标签，回滚就是 `mv`

## 想看更深？

跑出来的 DPO 数据是真实可用的微调输入——`core/data/corpora/*/dpo.jsonl` 可以喂给 TRL / LLaMA-Factory 等下游训练器，做真正的权重更新。

更详细的设计文档在 `references/architecture.md`，跟 Claude 说"解释下架构"它就会按需读。
