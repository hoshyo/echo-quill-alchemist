# 续写相似度评分细则（评分孙子 agent 必读）

本文件是评分孙子 agent 的唯一打分依据。读完本文 + 真实本章 + 生成本章后开始评分。

## 总览：六维加权

| 轴 | 权重 | 名称 | 评估对象 |
|--|--|--|--|
| style     | 0.25 | 语言风格 | 句长分布、修辞密度、画面感、书面/口语比 |
| plot      | 0.20 | 情节连贯 | 是否承接前章悬念、是否走在合理情节方向 |
| character | 0.20 | 人物刻画 | 角色行为、对白、心理是否符合人设 |
| tone      | 0.15 | 节奏与基调 | 紧张/松弛、悲喜、冷暖、章末情绪 |
| world     | 0.10 | 世界观一致 | 设定、术语、规则的前后一致性 |
| diction   | 0.10 | 用词遣句 | 高频词、忌讳词、专属词使用 |

`overall_similarity = sum(axis.score * axis.weight)`，结果保留四位小数。

## 各轴评分细则

### 1. style（语言风格）— 权重 0.25

**子项**（每子项各占本轴 1/4，先各自打 0-1，再平均得本轴 score）：

- **1.1 句长分布**：把生成稿与真实稿各拆 200 句样本，比较句长直方图（短 < 12 字 / 中 12-25 / 长 > 25）。三档比例差 ≤ 5pp 给 1.0；每超 5pp 扣 0.2
- **1.2 修辞密度**：明喻/暗喻/拟人/排比每千字出现次数差异。差 ≤ 1 次/千字 给 1.0；每超 1 次扣 0.2
- **1.3 画面感**：感官描写（视/听/嗅/味/触）密度。生成稿与真实稿比例差 ≤ 10% 给 1.0；每超 10% 扣 0.2
- **1.4 书面/口语比**：叙述与对白的语体差异（如使用"何/此/乃"等文言虚词的频率）。差 ≤ 10% 给 1.0；每超 10% 扣 0.2

### 2. plot（情节连贯）— 权重 0.20

**子项**：

- **2.1 开场承接**：生成稿首段是否接续前章末段的场景 / 情绪 / 时间线。完美承接 1.0；只承接其中两项 0.7；只承接一项 0.4；完全脱节 0
- **2.2 中段推进**：中段事件是否落在前章伏笔指向的方向。
  - 真实稿写了 N 个关键情节点，生成稿命中 M 个 → score = M / N
  - 注意：方向相同即可，不要求字面相同。例：真实稿"主角去找 A 谈判"，生成稿"主角去找 A 寻求结盟"算命中
- **2.3 章末钩子**：章末是否抛出新悬念 / 未尽情绪。三档：抛出且与真实稿同方向 1.0；抛出但方向不同 0.5；没抛出钩子 0

### 3. character（人物刻画）— 权重 0.20

**子项**：

- **3.1 行为一致**：本章主要角色（出场 ≥ 2 次的角色）的行动决策是否符合此前章节建立的人设。每个角色单独评 0-1，最后取均值
- **3.2 对白一致**：核心角色对白的句式 / 用词 / 语气是否与前章保持一致。同上每角色评 0-1 取均值
- **3.3 心理刻画**：心理描写的视角和深度是否一致（POV 主角内心戏密度、配角是否始终外视角等）

### 4. tone（节奏与基调）— 权重 0.15

**子项**：

- **4.1 整章基调**：本章主导情绪（悲/喜/紧张/松弛/温暖/冷峻）是否与真实稿一致。完全一致 1.0；色调相似但强度不同 0.6；完全相反 0
- **4.2 节奏曲线**：本章前/中/后三段的节奏（动作密度、对白密度、内心戏密度）是否与真实稿同形。三段都对 1.0；两段对 0.6；一段对 0.3

### 5. world（世界观一致）— 权重 0.10

**子项**：

- **5.1 设定无错**：生成稿是否引入了违反前章设定的元素（如已建立的物理规则、历史背景、社会结构）。出现违反则该项扣 0.3 / 处，最低 0
- **5.2 术语正确**：专有名词（地名、人名、组织名、法术名等）拼写与用法正确。每错一处扣 0.2，最低 0
- **5.3 时空连贯**：生成稿场景的时间地点切换是否与前章衔接合理

### 6. diction（用词遣句）— 权重 0.10

**子项**：

- **6.1 高频词覆盖**：从前章及种子章节统计的"作者偏好高频实词 Top 30"（先验列表可由评分子 agent 自己快速统计）在生成稿中的命中率。命中率 ≥ 70% 给 1.0；每低 10pp 扣 0.2
- **6.2 忌讳词规避**：作者明显避免的词（如某体裁里几乎不出现的现代俗语 / AI 风套话："总而言之 / 不可否认 / 值得注意的是"）是否被生成稿误用。每出现一处扣 0.3
- **6.3 专属词风**：作者特色搭配（如某些独有的形容词+名词搭配）是否被复现。出现 1+ 处 1.0；0 处 0.5

## 噪声忽略清单（不计入差异）

下列差异**不**扣任何轴的分：

- 数字 / 日期格式细节（"二〇二三年" vs "2023 年"）
- 引号样式（"" vs 「」）
- 段落数量差（在总字数差 ≤ 30% 的前提下）
- 同义词替换（"望" vs "看"，在密度匹配的前提下）

## 归因要求（写到 report.md）

每个轴扣分时必须写明**至少一条归因**。归因落在四选一：

1. **skill 缺陷**：当前 SKILL.md 没说 / 说得太软 / 规则模糊
2. **skill 误导**：SKILL.md 现有条款把生成 agent 带偏了
3. **生成 agent 失误**：skill 已说清楚但生成 agent 没跟上（罕见，多次出现需调整 skill 加强 / 加例子）
4. **不可避免**：作者真实下一章里出现了**前章没有任何线索**的情节转折（如全新角色突然登场），无法依靠 skill 和前章信息预测——这条不算 skill 缺陷，但可在 report 末尾的"informational"列表中注明

注意：归因只用于改 skill 孙子 agent 的下一步修补；不影响本轴打分本身。

## score.json 字段约束

```json
{
  "chapter": <int>,
  "attempt": <int>,
  "axes": {
    "style":     { "score": <float 0-1>, "weight": 0.25, "weighted": <float> },
    "plot":      { "score": <float 0-1>, "weight": 0.20, "weighted": <float> },
    "character": { "score": <float 0-1>, "weight": 0.20, "weighted": <float> },
    "tone":      { "score": <float 0-1>, "weight": 0.15, "weighted": <float> },
    "world":     { "score": <float 0-1>, "weight": 0.10, "weighted": <float> },
    "diction":   { "score": <float 0-1>, "weight": 0.10, "weighted": <float> }
  },
  "overall_similarity": <float 0-1>,
  "top_gaps": [<axis-name-1>, <axis-name-2>],
  "notes": "<一句话总评>"
}
```

字段校验：
- 每轴 `score` ∈ [0.0, 1.0]
- `weighted = score * weight`，浮点误差 ≤ 1e-4
- `overall_similarity = sum(axes[*].weighted)`，浮点误差 ≤ 1e-4
- `top_gaps` 是按 `(1 - score) * weight` 降序排列后的前 2 个轴名（最值得改的两维）
